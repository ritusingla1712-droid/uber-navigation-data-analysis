# 🚗 Uber Navigation Insights Analysis (SQL Project)

## 📌 Project Overview

This project analyzes Uber ride, driver, and passenger data using SQL to uncover insights related to ride demand, pricing, driver performance, and operational efficiency.

The analysis is structured across **basic and intermediate SQL queries**, showcasing strong command over real-world data problem-solving.

---

## 🎯 Objectives

* Understand ride distribution and demand patterns
* Analyze driver earnings and performance
* Evaluate fare trends and ride efficiency
* Extract insights using structured SQL queries

---

## 🗂️ Dataset Tables

The project uses three main tables:

* **ride1** → ride details (fare, distance, duration, locations, payment)
* **driver1** → driver information (earnings, ratings)
* **passenger1** → passenger details

---

## 🧠 SQL Concepts Applied

* Data filtering using `WHERE`
* Aggregations using `GROUP BY`
* Sorting using `ORDER BY`
* Subqueries for advanced filtering
* Joins between multiple tables
* Conditional analysis
* Percentage calculations

---

## 📊 Key Analysis Performed

### 🔹 Ride Analysis

* Total number of rides calculated
* Most common pickup locations identified
* Peak demand locations analyzed
* Rides filtered based on fare, duration, and distance

### 🔹 Driver Insights

* Top drivers based on earnings
* Average driver ratings
* Drivers handling high-distance trips
* Drivers completing 100+ rides

### 🔹 Passenger Insights

* Passengers with highest ride frequency
* Top spenders identified
* Ride behavior across multiple locations

### 🔹 Payment & Pricing

* Ride distribution by payment method
* Cash vs Credit usage patterns
* Average fare calculations
* Fare comparison with overall average

---

## 📈 Sample Insights

* Certain pickup locations dominate ride demand
* High-distance rides are handled by a limited set of drivers
* Payment methods show clear usage patterns across locations
* Fare distribution varies significantly with ride distance
